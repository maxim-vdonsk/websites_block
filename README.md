# Блокировщик сайтов с расписанием

Программа с графическим интерфейсом для блокировки сайтов по расписанию или вручную.
Блокировка работает через системный файл `hosts` — браузер не может открыть сайт, потому что его адрес перенаправляется на `127.0.0.1` (локальный компьютер).

> **Автор:** maxim_vdonsk
> **Репозиторий:** https://github.com/dansemaxtor/websites_block

---

## Как это работает (коротко)

1. Программа записывает в системный `hosts`-файл строки вида:
   ```
   127.0.0.1 youtube.com
   127.0.0.1 www.youtube.com
   ```
2. Операционная система перестаёт пускать браузер на эти сайты.
3. При разблокировке программа удаляет только свои строки — остальное содержимое `hosts` не трогается.

## Технологии

| Библиотека | Зачем |
|---|---|
| `tkinter` | Графический интерфейс (встроена в Python) |
| `sqlite3` | Хранение паролей и списка сайтов (встроена в Python) |
| `schedule` | Запуск задач по расписанию (устанавливается через pip) |
| `hashlib` | Хэширование пароля через SHA-256 (встроена в Python) |
| `threading` | Планировщик работает в фоне, не замораживая GUI (встроена в Python) |
| `platform` | Определение ОС для выбора пути к hosts-файлу (встроена в Python) |

## Требования

- Python **3.10** или новее (используется синтаксис `str | None`)
- Права **администратора / root** для изменения файла `hosts`

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/dansemaxtor/websites_block.git
cd websites_block
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Запустить программу

> **Важно:** программа изменяет системный файл `hosts`, поэтому нужны права администратора.

**macOS / Linux:**
```bash
sudo python main.py
```

**Windows:**
Щёлкните правой кнопкой по `main.py` → «Запустить от имени администратора»,
или в командной строке (запущенной от администратора):
```cmd
python main.py
```

## Первый запуск — создание пароля

При первом открытии программа попросит придумать пароль.
Введите любой пароль и нажмите **«Войти»** — он сохранится в зашифрованном виде.
После этого программа попросит перезапустить себя.

При следующих запусках нужно будет вводить этот же пароль.

## Как пользоваться

### Добавление сайтов

В поле **«Сайты для блокировки»** введите домены — по одному на строку:

```
youtube.com
vk.com
instagram.com
```

> Не нужно писать `https://` или `www.` — программа добавит `www.` сама.

---

### Заблокировать сейчас

Нажмите кнопку **«Заблокировать сейчас»** — сайты немедленно перестанут открываться.

---

### Блокировка по расписанию

1. Введите время начала и окончания в полях **«Время начала»** и **«Время окончания»** в формате `ЧЧ:ММ` (например, `09:00` и `18:00`).
2. Нажмите **«Запуск по расписанию»**.
3. Программа должна оставаться открытой — расписание работает, пока она запущена.

---

### Разблокировать сайты

Нажмите **«Разблокировать всё»** — список очистится, и сайты снова откроются.

## Частые проблемы

### Сайты не блокируются
- Убедитесь, что программа запущена **от администратора / sudo**.
- Очистите кэш DNS:
  - macOS: `sudo dscacheutil -flushcache`
  - Linux: `sudo systemctl restart systemd-resolved`
  - Windows: `ipconfig /flushdns`
- Перезапустите браузер после блокировки.

### Ошибка «Permission denied»
- Программа не может изменить файл `hosts` без прав администратора.
- Запустите через `sudo` (macOS/Linux) или от имени администратора (Windows).

### Забыл пароль
- Удалите файл `app_data.db` — при следующем запуске программа создаст новый пароль.
  **Внимание:** список заблокированных сайтов тоже удалится.

---

## Безопасность

- Пароль хранится в базе данных в виде SHA-256 хэша — исходный пароль нигде не сохраняется.
- База данных `app_data.db` находится рядом со скриптом на вашем компьютере.
